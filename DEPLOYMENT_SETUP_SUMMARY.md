# Deployment Setup Summary

## Files Created

### 1. `.circleci/config.yml`
CircleCI pipeline configuration with 3 jobs:
- **lint-and-test**: Runs on every push, checks Python syntax
- **build-docker**: Builds Docker image on main/develop branches
- **deploy-sandbox**: Deploys to sandbox server on main branch

### 2. `Dockerfile`
Docker image configuration:
- Based on official Odoo 17 image
- Copies custom addons from `./addons`
- Sets up health checks
- Exposes port 8069

### 3. `docker-compose.yml`
Local and production Docker composition:
- PostgreSQL 15 database service
- Odoo service with volume mounts
- Environment variable support
- Health checks for both services

### 4. `.dockerignore`
Excludes unnecessary files from Docker build context:
- Git files
- Markdown documentation
- Test files
- IDE configuration
- Python cache files

### 5. `.env.example`
Template for environment configuration:
- Database credentials
- Odoo admin password
- MercadoPago settings
- Application environment variables

### 6. `.gitignore`
Prevents committing sensitive files:
- `.env` files (credentials)
- IDE configuration
- Python cache
- Docker logs
- SSH keys

### 7. `scripts/setup-server.sh`
Automated server setup script:
- Installs Docker and Docker Compose
- Creates `deploy` user
- Generates SSH keys
- Sets up directory structure
- Configures sudo permissions

### 8. `CIRCLECI_DEPLOYMENT.md`
Complete deployment guide:
- Architecture diagram
- Step-by-step instructions
- CircleCI setup
- Server provisioning
- Environment configuration
- Troubleshooting section
- Security notes

### 9. `DEPLOYMENT_QUICKSTART.md`
Quick reference (5 minutes):
- TL;DR commands
- Step-by-step walkthrough
- MercadoPago configuration
- Common troubleshooting

## Deployment Flow

### Local Development
```bash
# Make changes
nano addons/br_mercadopago/models/br_payment.py

# Test locally
docker-compose up -d
# Visit http://localhost:8069

# Commit
git add .
git commit -m "Feature: add payment method filtering"
```

### Automated Deployment
```bash
git push origin main
    ↓
CircleCI triggers automatically:
    1. Lint checks (Python syntax)
    2. Build Docker image
    3. Deploy to sandbox server
    ↓
Your server updates within 5 minutes
    ↓
http://YOUR_SERVER_IP:8069 has latest code
```

## Required Configuration

### CircleCI Context Variables
Set these in CircleCI Project Settings → Contexts → `botonrojo-deploy-sandbox`:

| Variable | Example |
|---|---|
| `SANDBOX_HOST` | `34.39.181.0` or `deploy.example.com` |
| `SSH_PRIVATE_KEY_SANDBOX` | (Generated by setup-server.sh) |

### Server Environment (.env)
Create on deployment server at `/opt/botonrojo/.env`:

```env
DB_PASSWORD=secure_database_password
ADMIN_PASSWORD=secure_odoo_admin_password
ENVIRONMENT=sandbox
DEBUG=0
```

## Next Steps

1. **Push to GitHub**: Ensure your code is in the repository
2. **Run server setup**: Execute `scripts/setup-server.sh` on your server
3. **Add CircleCI variables**: Configure `SANDBOX_HOST` and `SSH_PRIVATE_KEY_SANDBOX`
4. **Create .env**: Set up environment configuration on server
5. **Trigger deployment**: Push to main branch
6. **Verify**: Check CircleCI dashboard and visit your server URL
7. **Configure MercadoPago**: Follow SETUP_GCP_SIMPLE.md or DEPLOYMENT_QUICKSTART.md
